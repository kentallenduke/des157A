Process work: Developing MadLibs 2.0

For the final project, I am improving the functionality of the Flip button for my Mad Libs project. One common complaint was that users playing the word game can flip the index card even if they did not fill out all of the input fields. This introduced confusion since they were unsure if they needed to fill out one or all of the inputs. To fix this, I plan to implement a conditional check on the button. Upon page load, the button will be disabled by default. The browser will need to verify if the player filled out all of the input fields. Additionally, I need to write code that stops the browser from verifying the fields on the back side of the card. When the user fills out all of the fields, the browser will enable the button, allowing the user to flip the card and view the story.
